Introdução a API Clima Music

O objetivo desta documentação é orientar o desenvolvedor sobre como integrar com a API do Clima Music, API que irá disponibilizar músicas de acordo com a temperatura de uma cidade e gerador de métricas das requisições, descrevendo os serviços disponíveis com exemplos de requisição e respostas.

Todas as operações requerem credenciais de acesso (Client ID e Client Secret) específicos para respectivos para comunicação com a API Spotify e key para a API externa OpenWeatherMaps, mas não se preocupe, essas credenciais são mantidas e manipuladas do nosso lado.

Arquitetura

A integração é realizada através de serviços disponibilizados como Web Services. O modelo utilizado é bem simples e fácil de ser utilizado, leia abaixo as especificações dos endpoints disponíveis nesta API que são comunicáveis através do protocolo HTTP.

GET - O método HTTP GET é utilizado para consultas de recursos já existentes. Por exemplo, músicas com base na temperatura da cidade e estatísticas das requsições já realizadas.

Instalação

O nosso serviço é disponibilizado via API REST a qual não necessita de nenhuma instalação, basta você consumir os dados disponibilizados em nossos endpoints desta documentação.

APIs

Nesta secão você irá encontrar todos os endpoints disponíveis na API Clima Music e suas respectivas descrições.

Nós também disponibilizamos uma collection do Postman que você poderá baixar e utilizar em seus testes.

GET api/weather/{city}

Este endpoint irá disponibilizar músicas cujo gênero irá variar de acordo com a temperatura da cidade.
A cada requisição será retornado um response aleatório de músicas do mesmo gênero.

Parâmetros: city - Nome da cidade (obrigatório)


Exemplos de request:

PYTHON 
							

import requests
import json

url = 'http://climamusic.kinghost.net/api/weather/campinas'
headers = {
	'Content-Type': 'application/json',
	'Accept': 'application/json'
}
response = requests.request('GET', url, headers=headers)
response.json()
BASH 
							

curl -X GET \
	-G "http://climamusic.kinghost.net/api/weather/campinas" \
	-H "Content-Type: application/json" \
	-H "Accept: application/json"
JAVASCRIPT 
							

const url = new URL(
    "http://climamusic.kinghost.net/api/weather/campinas"
);

let headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers: headers,
})
    .then(response => response.json())
    .then(json => console.log(json));
PHP 
							

$client = new \GuzzleHttp\Client();
$response = $client->get(
	'http://climamusic.kinghost.net/api/weather/campinas',
	[
		'headers' => [
			'Content-Type' => 'application/json',
			'Accept' => 'application/json',
		],
	]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

Integrações

A nossa API Clima Music utiliza API'S de terceiros para que possamos realizar todas as nossas operações e processos de maneira precisa e eficiente.

Segue abaixo as respectivas API'S.

OpenWeatherMap API

O OpenWeatherMaps ajuda a encontrar informações, links e documentos úteis para começar a usar nossos serviços de API de metereologia sem dificuldades.

O OpenWeatherMaps é um dos principais fornecedores de informações meteorológicas digitais.

Spotify API

Com base em princípios REST simples, os endpoints da API da Web do Spotify retornam metadados JSON sobre artistas, álbuns e faixas diretamente do Catálogo de dados do Spotify.

FAQs

Dúvida Frequentes

Eu preciso realizar alguma método de autenticação para acessar os endpoints da API REST?

Não, a nossa API é totalmente aberta ao uso.

Estou recebendo erros ao usar as APIs REST. O que eu faço?

Se você esta recebendo um response com um erro, favor anote o código do erro e veja a descrição do mesmo em nossa tabela de erros descritos da API Clima Music, clique AQUI para acessar

O que é uma API REST?

API REST é uma interface que fornece dados em um formato padronizado baseado em requisições HTTP.

Como começo a usar a API REST Clima Music?

Como não é necessario autenticação em nossa API, basta você realizar a requisição adequada para cada endpoint de nossa API, passando os parâmetros necessários e você receberá um response como resposta.

Para usar esta API eu preciso realizar algum pagamento ou cadastro?

Não, nossa API é livre de qualquer custo e cadastro exigido ao usuário.
